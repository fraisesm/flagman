# Роли пользователей

## Три роли

| Роль | Значение | Описание |
|------|----------|----------|
| `admin` | Администратор | Видит всё: всех пользователей, все документы, статистику. Управляет системой. |
| `boss` | Начальник | Загружает документы и отправляет их на подпись сотрудникам. |
| `employee` | Сотрудник | Получает документы, подписывает или пересылает дальше. |

## Создание первого admin

После запуска сервера выполни один раз:

```bash
cd back
python create_admin.py
```

Это создаст аккаунт:
- **Email:** `admin@flagman.ru`
- **Пароль:** `admin123`

Потом смени пароль через `PUT /auth/me`.

## Регистрация пользователей

```http
POST /auth/register
Content-Type: application/json

{
  "full_name": "Иван Иванов",
  "email": "ivan@company.ru",
  "phone": "+79001234567",
  "password": "password123",
  "role": "employee"   // или "boss", "admin"
}
```

**Минимальная длина пароля:** 6 символов.

## Логин — ответ содержит роль

```http
POST /auth/login

→ {
  "access_token": "...",
  "token_type": "bearer",
  "user_id": 1,
  "full_name": "Иван Иванов",
  "email": "ivan@company.ru",
  "role": "employee"
}
```

Фронт сохраняет `role` и показывает нужный интерфейс.

## Миграция базы данных

Если база уже существует, нужно добавить колонку `role`:

```sql
ALTER TABLE users ADD COLUMN role VARCHAR DEFAULT 'employee' NOT NULL;
```

Либо удали файл `*.db` и перезапусти сервер — таблицы создадутся заново.
